## Critical Blocker — INFRA-10

### Problem

`src/bunny_cdn_sdk/__init__.py` lines 19–21:

```python
if TYPE_CHECKING:
    from bunny_cdn_sdk.core import CoreClient
    from bunny_cdn_sdk.storage import StorageClient
```

`TYPE_CHECKING` is `False` at runtime — it is only `True` when mypy/pyright/ty runs. This guard was placed in Phase 2 (02-03-PLAN) as a forward-reference placeholder because CoreClient and StorageClient did not exist yet. Phase 3 implemented the modules but never removed the guard.

### Impact

```
$ python -c "from bunny_cdn_sdk import CoreClient"
ImportError: cannot import name 'CoreClient' from 'bunny_cdn_sdk'
```

The SDK's core value — "instantiate a client with their API key" — is blocked on the first import. The ROADMAP Phase 2 success criterion (`from bunny_cdn_sdk import CoreClient, StorageClient, BunnyAPIError` resolves without ImportError) is not met.

### Fix (1 line change)

In `src/bunny_cdn_sdk/__init__.py`, replace:

```python
from typing import TYPE_CHECKING
...
if TYPE_CHECKING:
    from bunny_cdn_sdk.core import CoreClient
    from bunny_cdn_sdk.storage import StorageClient
```

With:

```python
from bunny_cdn_sdk.core import CoreClient
from bunny_cdn_sdk.storage import StorageClient
```

(Remove the `from typing import TYPE_CHECKING` import too — it will be unused.)

## Integration Findings

### CRITICAL: INFRA-10 — Public surface broken

See above. `from bunny_cdn_sdk import CoreClient/StorageClient` fails at runtime.

### WARNING: Tests bypass public surface

All 58 tests import directly from submodules (`bunny_cdn_sdk.core`, `bunny_cdn_sdk.storage`) rather than from the public package. This means the INFRA-10 breakage is invisible to the test suite. A future regression to `__init__.py` would not be caught.

**Recommended fix:** Add one smoke test in `conftest.py` or a dedicated `test_public_surface.py`:
```python
def test_public_imports():
    from bunny_cdn_sdk import CoreClient, StorageClient, BunnyAPIError
    assert CoreClient is not None
    assert StorageClient is not None
```

### WARNING: Context manager lifecycle untested (INFRA-04/INFRA-05)

`_client.py` shows 82% coverage (10 uncovered lines). The uncovered paths are the sync/async context manager methods — the cleanup path where `_client_owner=True` triggers `aclose()`. These are functional (verified manually during Phase 2) but have no regression protection in the test suite.

### WARNING: list_single_page orphaned (INFRA-08)

`_pagination.py` exports `list_single_page()` in `__all__`, but `core.py` implements all `list_*()` methods by calling `_sync_request()` directly — the helper is never used. INFRA-08's intent (single-page fetch) is met by the direct approach, but the exported function is dead code. Minor inconsistency, not a bug.

### INFO: PaginatedResponse not on public surface (INFRA-07)

`PaginatedResponse` TypedDict is internal to `_types.py`. Users cannot `from bunny_cdn_sdk import PaginatedResponse` to type-annotate their own code. Not a stated requirement, but a usability gap for typed applications.
