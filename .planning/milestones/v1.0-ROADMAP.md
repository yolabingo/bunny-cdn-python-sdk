## Phase 2: Base Client Infrastructure

**Goal:** Deliver the `_BaseClient`, `_types.py`, `_pagination.py`, and `__init__.py` so the full internal machinery — auth injection, error mapping, concurrent gather, pagination, and context manager — is complete before any service client is written.

**Plans:** 3/3 complete

- [x] 02-01-PLAN.md — Implement `_client.py` with `httpx.AsyncClient` internals, `_request()` chokepoint, `_gather()`, context manager support
- [x] 02-02-PLAN.md — Implement `_types.py` (`PaginatedResponse` TypedDict) and `_pagination.py` (callback-based iterator)
- [x] 02-03-PLAN.md — Implement `__init__.py` re-exports (initially with TYPE_CHECKING guard; fixed post-audit)

**Requirements:** INFRA-01, INFRA-02, INFRA-03, INFRA-04, INFRA-05, INFRA-07, INFRA-08, INFRA-10

**Outcome:**
- `_BaseClient` wraps `httpx.AsyncClient`; sync surface via `asyncio.run()`; optional transport injection
- Every request carries `AccessKey` + `User-Agent: bunny-cdn-sdk/0.1.0` headers
- `_gather([coro1, coro2])` returns results in input order, concurrently
- `PaginatedResponse` TypedDict: `Items`, `CurrentPage`, `TotalItems`, `HasMoreItems`
- `pagination_iterator()` auto-paginates via callback; `list_single_page()` single-page fetch
- `from bunny_cdn_sdk import CoreClient, StorageClient, BunnyAPIError` resolves without error
- Post-audit INFRA-10 fix: TYPE_CHECKING guard removed (commit `5b57a66`)
- Key commits: `303425d`, `644c7e3`, `98430cc`, `9cd9cc9`, `5b57a66`

## Phase 3: Core API & Storage API Clients

**Goal:** Implement `CoreClient` and `StorageClient` so the SDK delivers its full v1 surface.

**Plans:** 2/2 complete

- [x] 03-01-PLAN.md — Implement CoreClient with all 27+ methods across 5 resource groups
- [x] 03-02-PLAN.md — Implement StorageClient with upload, download, delete, list and 10-region URL mapping

**Requirements:** CORE-01 through CORE-11, STOR-01 through STOR-05

**Outcome:**
- `CoreClient` — 37 public methods: Pull Zones (12), Storage Zones (6), DNS Zones (9), Video Libraries (5), Utilities (5)
- `get_pull_zones([id1, id2])` issues concurrent requests via `asyncio.gather`; returns results in input order
- `iter_pull_zones()` transparently fetches page 2+ when `HasMoreItems` is true
- `StorageClient` — upload/download/delete/list; 10-region HTTPS base-URL map (falkenstein + de, ny, la, sg, syd, uk, se, br, jh)
- `ValueError` raised at construction time for unknown region
- HTTP Basic Auth (`base64(zone_name:password)`) on every Storage request
