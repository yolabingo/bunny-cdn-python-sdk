## Milestone Summary

**Key Decisions:**

| Decision | Outcome |
|----------|---------|
| UserWarning (not ValueError) on client= + max_retries conflict | Clean API — no surprise exceptions; warning is sufficient |
| `max_retries=0` default preserves v1.0 behavior | Zero-config backward compat guaranteed |
| `pragma: no cover` on unreachable post-loop sentinel | Correct 100% coverage without gaming metrics |
| `[tool.ruff.lint.per-file-ignores]` for tests/** | Consistent lint treatment across all test files |
| `client=` made keyword-only on CoreClient and StorageClient | Cleaner API; positional use was always ambiguous |

**Issues Resolved:**

- Context manager aclose() path was untested (v1.0 tech debt) — covered by lifecycle tests
- `list_single_page` was orphaned export — removed cleanly (zero callers confirmed)
- `random.uniform` jitter flagged by ruff S311 — resolved with `# noqa: S311` (not a crypto use)
- `httpx.AsyncClient` has no public `.transport` — corrected to `._transport` in test code

- `test_public_surface.py` missing `RetryTransport` assertion
- 2 pre-existing `ty` errors in `storage.py` (carried from v1.0)
- 59 pre-existing ruff violations across codebase (carried from v1.0)

**Technical Debt Incurred:**
