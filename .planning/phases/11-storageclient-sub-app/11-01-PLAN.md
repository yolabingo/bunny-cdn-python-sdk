**Module-level (no State/StorageClient import at module level — circular import prevention):**
```python
"""Storage file CLI sub-app."""

from __future__ import annotations

import os
from typing import cast

import typer

from bunny_cdn_sdk.cli._output import err_console, output_result, sdk_errors

storage_app = typer.Typer(no_args_is_help=True, help="Manage Bunny CDN storage files.")
_COLUMNS = ["ObjectName", "Length", "IsDirectory", "LastChanged"]
```

**Auth guard helper (inline in each command — do not extract to a function that would need State at module level):** Each command does the same check:
```python
state = cast("State", ctx.obj)
if not state.zone_name or not state.storage_key:
    err_console.print(
        "Missing storage auth. Use --zone-name/--storage-key "
        "or set BUNNY_STORAGE_ZONE/BUNNY_STORAGE_KEY."
    )
    raise typer.Exit(1)
```

**StorageClient construction (inside each command body):**
```python
client = StorageClient(state.zone_name, state.storage_key, region=state.region)
# NOTE: second positional arg maps to `password` param — correct usage above
```

**list command** (`@storage_app.command("list")`):
- Signature: `def list_files(ctx: typer.Context, path: str = typer.Argument("/", help="Storage path to list (default: zone root)")) -> None`
- Docstring: `"""List files and directories at a storage path."""`
- Deferred imports: `from bunny_cdn_sdk.cli._app import State` + `from bunny_cdn_sdk.storage import StorageClient`
- Body: auth guard → `with sdk_errors():` → construct client → `files = client.list(path)` → `output_result(files, columns=_COLUMNS, json_mode=state.json_output)`

**upload command** (`@storage_app.command("upload")`):
- Signature: `def upload_file(ctx: typer.Context, local_path: str = typer.Argument(..., help="Local file path to upload"), remote_path: str = typer.Argument(..., help="Destination path in storage zone")) -> None`
- Docstring: `"""Upload a local file to storage."""`
- Deferred imports: `from bunny_cdn_sdk.cli._app import State` + `from bunny_cdn_sdk.storage import StorageClient`
- Body inside `with sdk_errors():`:
  1. `if not os.path.isfile(local_path): raise ValueError(f"Local file not found: {local_path!r}")` — raises ValueError caught by sdk_errors()
  2. `client = StorageClient(state.zone_name, state.storage_key, region=state.region)`
  3. `with open(local_path, "rb") as fh: client.upload(remote_path, fh.read())`
  4. `typer.echo(f"Uploaded {local_path!r} -> {remote_path!r}")`

**download command** (`@storage_app.command("download")`):
- Signature: `def download_file(ctx: typer.Context, remote_path: str = typer.Argument(..., help="Storage path to download"), local_path: str = typer.Argument(..., help="Local destination path")) -> None`
- Docstring: `"""Download a storage file to local filesystem. Overwrites local_path if it exists."""`
- Deferred imports: `from bunny_cdn_sdk.cli._app import State` + `from bunny_cdn_sdk.storage import StorageClient`
- Body inside `with sdk_errors():`:
  1. `client = StorageClient(state.zone_name, state.storage_key, region=state.region)`
  2. `data = client.download(remote_path)`
  3. Wrap write in try/except OSError: `try: \n    with open(local_path, "wb") as fh: fh.write(data)\nexcept OSError as exc:\n    raise ValueError(str(exc)) from exc`
  4. `typer.echo(f"Downloaded {remote_path!r} -> {local_path!r}")`

**delete command** (`@storage_app.command("delete")`):
- Signature: `def delete_file(ctx: typer.Context, remote_path: str = typer.Argument(..., help="Storage path to delete"), yes: bool = typer.Option(False, "--yes", "-y", help="Skip confirmation prompt")) -> None`
- Docstring: `"""Delete a file from storage (prompts for confirmation)."""`
- Deferred imports: `from bunny_cdn_sdk.cli._app import State` + `from bunny_cdn_sdk.storage import StorageClient`
- Body inside `with sdk_errors():`:
  1. `if not yes: typer.confirm(f"Delete {remote_path!r} from storage zone?", abort=True)`
  2. `client = StorageClient(state.zone_name, state.storage_key, region=state.region)`
  3. `client.delete(remote_path)`
  4. `typer.echo(f"Deleted {remote_path!r}.")`

**Critical constraints:**
- NEVER import `State` or `StorageClient` at module level — only inside command function bodies
- `StorageClient(state.zone_name, state.storage_key, region=state.region)` — positional args for zone_name and password; `region` as keyword
- Auth guard checks `state.zone_name` and `state.storage_key` only — never check `state.region` (it always has a default of "falkenstein")
- upload/download use `typer.echo()` for success messages, NOT `output_result()` (no tabular data)
- list uses `output_result(files, columns=_COLUMNS, json_mode=state.json_output)` — no `typer.echo()`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → StorageClient | `local_path`, `remote_path` are user-supplied strings passed to filesystem and SDK |
| CLI auth env → StorageClient constructor | `BUNNY_STORAGE_ZONE` / `BUNNY_STORAGE_KEY` pass to SDK as credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Tampering | upload: `local_path` arg | mitigate | `os.path.isfile(local_path)` check before open; rejects directories and non-existent paths |
| T-11-02 | Information Disclosure | upload/download: `state.storage_key` in output | mitigate | Never echo `state.storage_key` in any success or error message — only echo file paths |
| T-11-03 | Tampering | download: `local_path` write destination | accept | User controls the local path — overwrite is expected CLI behavior (same as `curl -o`); no mitigation needed |
| T-11-04 | Tampering | remote_path traversal | accept | `StorageClient._build_url()` strips leading slash and prefixes zone_name — no traversal above zone root is possible |
| T-11-05 | Elevation of Privilege | Missing auth guard | mitigate | Auth guard checks `state.zone_name` and `state.storage_key` before constructing StorageClient — exits 1 with stderr message |
| T-11-06 | Denial of Service | Large file download into memory | accept | CLI use case; full-buffer is documented as out of scope for progress bars; mitigated in SDK layer via RetryTransport |
</threat_model>
