    ```python
    def output_result(
        data: Any,
        *,
        columns: list[str] | None = None,
        json_mode: bool = False,
    ) -> None:
        """Emit command output — JSON if json_mode, Rich table otherwise."""
        if json_mode:
            typer.echo(json.dumps(data, indent=2, default=str))
            return

        # Passthrough for plain strings and other non-dict/list scalars
        if not isinstance(data, (dict, list)):
            typer.echo(str(data))
            return

        # Normalize single dict to list so all rendering uses the same code path
        rows: list[dict[str, Any]] = [data] if isinstance(data, dict) else data  # type: ignore[assignment]

        if not rows:
            # Empty list — print empty table with headers if columns provided, else nothing
            if columns:
                table = Table(*columns)
                console.print(table)
            return

        # Derive column order: explicit list takes priority; fallback to first row's keys
        col_names: list[str] = columns if columns is not None else list(rows[0].keys())

        table = Table(*col_names)
        for row in rows:
            table.add_row(*[_cell(row.get(col)) for col in col_names])

        console.print(table)
    ```

    Key points:
    - `from rich.table import Table` must be added to the existing import block at the top
    - `sdk_errors()`, `_cell()`, `console`, `err_console` are NOT modified
    - The `# type: ignore[assignment]` comment on the rows assignment handles `ty` narrowing:
      after the isinstance check `data` is `dict | list`; the assignment is safe at runtime
    - `row.get(col)` returns None for missing keys → `_cell(None)` → "" (correct per CONTEXT.md)
    - `console.print(table)` routes through the existing Rich Console singleton (stdout)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| API response → output_result() | dict/list data from Bunny CDN API passes through output_result; values are rendered as strings |
| output_result() → stdout/stderr | table output to stdout; errors to stderr; no mixing |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | output_result() / Rich table | accept | All API response fields are rendered as strings via _cell(); no filtering of sensitive fields at this layer. Phase 10 callers pass explicit columns=[] lists that curate what is displayed — column curation is the access-control mechanism for sensitive fields |
| T-09-02 | Tampering | json.dumps default=str | accept | default=str converts non-serializable values to their str() representation; this is intentional for OUT-06. No injection risk: output goes to stdout as a string, not into any downstream parser or storage |
| T-09-03 | Denial of Service | output_result() rendering very large lists | accept | CLI users request data from their own accounts; unbounded table rendering is bounded by API pagination. No server-side risk. Performance concern is user-visible only |
| T-09-04 | Information Disclosure | console vs err_console routing | mitigate | Errors print via err_console (stderr=True); success output via console (stdout). Separation enforced in sdk_errors() (already tested in Phase 08). output_result() uses console.print() for table output — correct channel |
</threat_model>
