|----------|-------------|
| pyproject.toml → build chain | Package metadata controls what gets installed; malformed or tampered metadata can cause dependency confusion |
| uv.lock → installed packages | Lock file pins transitive deps; an outdated or missing lock allows supply chain substitution |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-01-02 | Information Disclosure | py.typed marker | accept | PEP 561 marker is public by design; exposes only that the package is typed, no sensitive data |
| T-01-03 | Spoofing | uv_build build backend version pin | mitigate | `uv_build>=0.11.6,<0.12.0` already constrains the build backend to a narrow range; verify lock file is committed |
| T-01-04 | Tampering | uv.lock supply chain | mitigate | Commit uv.lock to version control; run `pip-audit` in CI to detect known-vulnerable transitive deps |
</threat_model>
