# Phase 07 Plan 01: Wire max_retries / backoff_base into Constructor Chain

## Objective

Add `max_retries: int = 0` and `backoff_base: float = 0.5` keyword-only parameters to
`_BaseClient`, `CoreClient`, and `StorageClient`. When `max_retries > 0` and no custom
`client=` is provided, `_BaseClient.__init__` auto-creates a `RetryTransport`-backed
`httpx.AsyncClient`. The `max_retries=0` default preserves exact v1.0 behavior — no
`RetryTransport` is instantiated and no behavior changes for existing callers.

**Purpose:** RETRY-04 — users should be able to enable retries via a simple constructor
kwarg rather than manually wiring `RetryTransport`.

**Output:** Three modified source files. No new files. No changes to `__init__.py`.

### Task 1: Update _BaseClient.__init__ with retry auto-wiring

**Read first:** `src/bunny_cdn_sdk/_client.py` lines 1-45

**Action:**

1. Add `import warnings` after the existing `import asyncio` line (line 5). The final import block top section becomes:
   ```python
   import asyncio
   import warnings
   from typing import TYPE_CHECKING, Any, Self
   ```

2. Add `from bunny_cdn_sdk._retry import RetryTransport` after the `_exceptions` import block (after line 18, before the `if TYPE_CHECKING:` block):
   ```python
   from bunny_cdn_sdk._retry import RetryTransport
   ```

3. Replace the entire `__init__` method (lines 27-44) with:
   ```python
   def __init__(
       self,
       api_key: str,
       client: httpx.AsyncClient | None = None,
       *,
       max_retries: int = 0,
       backoff_base: float = 0.5,
   ) -> None:
       """Initialize the base client.

       Args:
           api_key: API key for authentication
           client: Optional httpx.AsyncClient to use; if not provided, one is created
           max_retries: Number of retry attempts on 429/5xx/network errors (default 0 = no retry).
               Ignored when ``client=`` is provided — configure RetryTransport on your
               AsyncClient directly in that case.
           backoff_base: Base delay in seconds for exponential backoff (default 0.5).
               Only used when ``max_retries > 0`` and ``client`` is None.
       """
       self.api_key = api_key
       if client is not None:
           if max_retries > 0:
               warnings.warn(
                   "max_retries is ignored when client= is provided. "
                   "Configure RetryTransport on your AsyncClient directly.",
                   UserWarning,
                   stacklevel=2,
               )
           self._client = client
           self._client_owner = False
       elif max_retries > 0:
           transport = RetryTransport(
               httpx.AsyncHTTPTransport(),
               max_retries=max_retries,
               backoff_base=backoff_base,
           )
           self._client = httpx.AsyncClient(transport=transport)
           self._client_owner = True
       else:
           self._client = httpx.AsyncClient()
           self._client_owner = True
   ```

Note: `client` is NOT keyword-only in `_BaseClient.__init__` — it must remain positional-or-keyword to preserve backward compatibility with `conftest.py`'s `_BaseClient("key", client=async_client)` and `super().__init__(password, client)` call patterns. Only `max_retries` and `backoff_base` are keyword-only (after `*`).

## Threat surface

| Boundary | Description |
|----------|-------------|
| caller → `_BaseClient.__init__` | `max_retries` and `backoff_base` are user-supplied constructor inputs |
| `_BaseClient` → `RetryTransport` | Internal wiring — no user data flows here |

### STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Denial of Service | `_BaseClient.__init__` `max_retries` | mitigate | `max_retries` is forwarded unchanged to `RetryTransport._max_retries`. `RetryTransport` already bounds delay at 60s (cap in `_backoff_delay`). No additional validation needed — excessively large values (e.g. 100) increase latency, not attack surface. Document in docstring that reasonable values are 1-5. |
| T-07-02 | Denial of Service | `backoff_base` | accept | Float value directly passed to `RetryTransport`. Extreme values (e.g. 10000.0) result in the delay being capped at 60s by `RetryTransport._backoff_delay`. No injection risk. |
| T-07-03 | Information Disclosure | `warnings.warn` in `__init__` | accept | Warning message does not contain user data (api_key, url). Warns on logic conflict only. Appropriate. |
| T-07-04 | Spoofing | Injected `client=` | accept | `_BaseClient` never modifies the injected `httpx.AsyncClient`. The client owns its transport stack. Callers are responsible for configuring auth on injected clients; the SDK adds `AccessKey` per-request via `_request()`, not at client construction. No new trust boundary opened. |
| T-07-05 | Elevation of Privilege | Import of `RetryTransport` from `_retry.py` | accept | `_retry.py` does not import from `_client.py` — no circular import risk. Both modules are internal; import path is one-directional. |
