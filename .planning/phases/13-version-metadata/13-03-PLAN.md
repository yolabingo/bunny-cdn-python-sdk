  <action>
**Step 1: Add `twine` to dev dependency groups in pyproject.toml.**

Locate the `[dependency-groups]` section. Add `twine` to the `audit` group (it is a build quality tool, fitting alongside `pip-audit`):

```toml
audit = ["pip-audit", "twine"]
```

If the `audit` group does not exist, add it as a new dependency group and include it in the `dev` group:
```toml
audit = ["pip-audit", "twine"]
```
The `dev` group already includes `{ include-group = "audit" }` so twine will be available in the dev environment automatically.

**Step 2: Sync the dev environment so twine is installed.**

```bash
uv sync --all-groups
```

**Step 3: Clean any existing dist/ artifacts to ensure a fresh build.**

```bash
rm -rf dist/
```

**Step 4: Build the package.**

```bash
uv build
```

Expected output: `Successfully built dist/bunny_cdn_sdk-2.1.0-py3-none-any.whl` and `dist/bunny_cdn_sdk-2.1.0.tar.gz` (exact filenames depend on the version in pyproject.toml).

**Step 5: Run twine check.**

```bash
uv run twine check dist/*
```

Expected output for EACH artifact:
```
Checking dist/bunny_cdn_sdk-2.1.0-py3-none-any.whl: PASSED
Checking dist/bunny_cdn_sdk-2.1.0.tar.gz: PASSED
```

If `twine check` reports any warning or error, investigate and fix before marking this task done. Common issues and fixes:
- "missing description" → ensure `readme = "README.md"` is in pyproject.toml `[project]` (done in Plan 01 Task 2)
- "invalid classifier" → verify classifiers match the full list from https://pypi.org/classifiers/
- "missing license" → ensure `license = { text = "MIT" }` is in pyproject.toml (done in Plan 01 Task 2)

Do NOT proceed to mark done if `twine check` reports any error or warning.
