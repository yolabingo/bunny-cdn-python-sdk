## Key Infrastructure from Phase 2

**_BaseClient interface:**
- `_sync_request(method, url, **kwargs)` — Synchronously makes HTTP request with auth injection and error mapping
- `_gather(*coroutines)` — Runs async operations concurrently and returns results in input order
- `api_key` property — Injected as `AccessKey` header
- Context manager support (`__enter__`, `__exit__`)

**Pagination:**
- `pagination_iterator(fetch_page: Callable, start_page: int)` — Async generator yielding individual items from paginated responses
- `list_single_page(fetch_page: Callable, page: int)` — Fetch single page without auto-pagination
- Both expect `fetch_page` to be async callable accepting page number

**Response format:**
- All methods return plain `dict` from `response.json()`
- Paginated responses have shape: `{"Items": [...], "CurrentPage": int, "TotalItems": int, "HasMoreItems": bool}`

## Base URL

All Core API calls use: `https://api.bunnycdn.com/`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User code → CoreClient | User supplies api_key; CoreClient trusts it is valid and kept secret |
| CoreClient → Bunny API | CoreClient injects api_key into Authorization header; Bunny validates it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | api_key parameter | mitigate | api_key is user-supplied string; SDK passes directly via AccessKey header per _BaseClient design (Phase 2). No validation of key format needed. |
| T-03-02 | Information Disclosure | Response body contains sensitive data (e.g., zone configs, IPs) | accept | Return raw dict from API; user is responsible for safe handling. SDK does not log or cache responses. |
| T-03-03 | Denial of Service | Concurrent batch fetch get_pull_zones([id1, id2, ...]) with large list | mitigate | asyncio.gather respects request/connection limits of underlying httpx.AsyncClient. Users should batch in reasonable chunks (~10-20 per batch). Document best practice in docstring. |
| T-03-04 | Elevation of Privilege | Custom hostname/IP add/remove operations | mitigate | Operations route through _BaseClient._sync_request which validates HTTP status and maps to exception hierarchy. API enforces zone ownership; SDK trusts API response. |
| T-03-05 | Tampering | HTTP MITM attack on requests | mitigate | Use HTTPS URLs (hardcoded base_url = "https://api.bunnycdn.com/"). httpx validates TLS certificates by default. |
| T-03-06 | Repudiation | Audit trail of operations | accept | SDK does not maintain operation logs. Bunny API provides audit trail. User can implement their own logging wrapper. |
</threat_model>
