**Step 0: Get live error list**

Use this output as the actual work queue. Fix each error category:

**E501 (line-too-long, > 100 chars):**
- For each flagged line, shorten by: wrapping long strings with implicit concatenation, splitting function args across lines, or breaking at natural punctuation
- Do NOT change variable names or logic to fit; restructure only formatting
- Example: `error_message = "This is a very long error message that exceeds the line limit"` → split with `\` or use parens

**TRY003 (raise-vanilla-args):**
- Extract exception messages to a variable before raise:
  ```python
  # Before:
  raise SomeError("Long inline message string")
  # After:
  msg = "Long inline message string"
  raise SomeError(msg)
  ```
- This is safe to apply uniformly to all TRY003 instances

**PLR2004 (magic-value-comparison in src/):**
- Replace magic values with named constants defined at module or function scope:
  ```python
  # Before:
  if status_code == 429:
  # After:
  HTTP_TOO_MANY_REQUESTS = 429
  if status_code == HTTP_TOO_MANY_REQUESTS:
  ```
- Note: PLR2004 in tests/ is already suppressed via per-file-ignores — only fix instances in src/

**E402 (module-level import order) — likely in _client.py:**
- Move any imports that appear after non-import statements to the top of the file
- If the import cannot be moved (circular import prevention is a STATE.md decision for cli/**), add to the appropriate per-file-ignores rather than restructuring

**DTZ011 / DTZ001 (datetime without timezone):**
- DTZ011 (`date.today()`): replace with `datetime.now(tz=timezone.utc).date()`
- DTZ001 (`datetime(...)` without tzinfo): add `tzinfo=timezone.utc` argument
- Add `from datetime import datetime, timezone` import if not present

**PTH123 / PTH113 (builtin open / os.path.isfile):**
- PTH123 (`open(path)`): replace with `pathlib.Path(path).open()`
- PTH113 (`os.path.isfile(path)`): replace with `pathlib.Path(path).is_file()`
- Add `from pathlib import Path` import if not present; remove `import os.path` if only used for isfile

**UP035 (deprecated typing imports):**
- `from typing import Callable` → `from collections.abc import Callable`
- `from typing import Iterator` → `from collections.abc import Iterator`
- `from typing import Generator` → `from collections.abc import Generator`
- Do not remove `TYPE_CHECKING` from typing — it is not deprecated

**ERA001 (commented-out code):**
- If the comment is clearly dead code: remove it entirely
- If it documents intent or a future consideration: convert to a proper `# NOTE:` or `# TODO:` comment, or remove and trust git history
- Do not leave ERA001-triggering comments in place

**BLE001 (blind-except):**
- Change `except Exception:` to the most specific exception type that makes sense
- If the code genuinely needs to catch all exceptions (last-resort handler), add `# noqa: BLE001` with a comment explaining why

**TRY300 (return in try block):**
- Move `return` from inside a `try:` block to the `else:` clause:
  ```python
  # Before:
  try:
      result = do_thing()
      return result
  except SomeError:
      handle()
  # After:
  try:
      result = do_thing()
  except SomeError:
      handle()
  else:
      return result
  ```

**PLR0913 (too-many-arguments in src/ — if any remain):**
- If the function is in cli/** code: add to cli/** per-file-ignores in pyproject.toml (already there for CLI, may need to extend)
- If in core SDK code: restructure using a config dataclass or kwargs dict

**PT001 (pytest fixture parentheses):**
- Change `@pytest.fixture()` to `@pytest.fixture` (remove empty parens)

**F841 (unused variable):**
- If the variable is intentionally unused (e.g., loop variable): prefix with `_`
- If it's genuinely unused: remove it
