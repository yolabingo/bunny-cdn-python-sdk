**`update` command (PZ-06, D-12, D-13, D-14, D-15):**

The update command parses `--set KEY=VALUE` pairs, fetches the resource before and after,
and displays only changed fields in a Field|Before|After table with red italic styling.

```python
@pull_zone_app.command("update")
def update_zone(
    ctx: typer.Context,
    id: int = typer.Argument(..., help="Pull zone ID"),
    set_: list[str] = typer.Option(
        [], "--set", help="Field to update as KEY=VALUE (repeatable)"
    ),
) -> None:
    """Update pull zone fields using --set KEY=VALUE."""
    from bunny_cdn_sdk.cli._app import State
    from bunny_cdn_sdk.core import CoreClient
    state = cast("State", ctx.obj)
    if not state.api_key:
        err_console.print("Missing API key. Use --api-key or set BUNNY_API_KEY.")
        raise typer.Exit(1)
    with sdk_errors():
        # Parse KEY=VALUE pairs (ValueError caught by sdk_errors)
        updates: dict[str, str] = {}
        for pair in set_:
            if "=" not in pair:
                raise ValueError(f"Invalid --set value: '{pair}' (expected KEY=VALUE)")
            k, v = pair.split("=", 1)
            updates[k] = v
        client = CoreClient(api_key=state.api_key)
        # D-12: fetch before state
        before = client.get_pull_zone(id)
        # Apply update
        after = client.update_pull_zone(id, **updates)
        # D-15: json mode outputs updated resource
        if state.json_output:
            output_result(after, json_mode=True)
            return
        # D-13, D-14: diff table with only changed rows in red italic
        changed_keys = [k for k in updates if str(before.get(k, "")) != str(after.get(k, ""))]
        if not changed_keys:
            typer.echo("No fields changed.")
            return
        table = Table("Field", "Before", "After")
        for k in changed_keys:
            table.add_row(
                k,
                Text(str(before.get(k, "")), style="bold red italic"),
                Text(str(after.get(k, "")), style="bold red italic"),
            )
        console.print(table)
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CLI args → CoreClient | User-supplied `--set KEY=VALUE` pairs and IDs flow into SDK API calls |
| env vars → State | `BUNNY_API_KEY` from environment flows into API authentication header |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | `--set` KEY=VALUE parsing | mitigate | Split on first `=` only; malformed pairs (no `=`) raise `ValueError` caught by `sdk_errors()` — exits 1 with actionable message |
| T-10-02 | Information Disclosure | API key in env var | accept | Standard env var pattern; key is never echoed to stdout or logged |
| T-10-03 | Elevation of Privilege | delete without confirmation | mitigate | D-01/D-02: `typer.confirm(abort=True)` by default; `--yes` is explicit opt-out |
| T-10-04 | Spoofing | Unvalidated zone ID type | accept | Typer `int` annotation rejects non-integer IDs before any SDK call |
</threat_model>
