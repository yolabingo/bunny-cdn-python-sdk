# Plan 02-03: Public Re-exports

**Objective**

Implement `src/bunny_cdn_sdk/__init__.py` to re-export:
- `CoreClient` and `StorageClient` (forward references to Phase 3 implementations)
- All exception classes from `_exceptions.py`

Maintain alphabetical ordering to satisfy ruff's RUF022 rule.

**Purpose**

The public API of `bunny-cdn-sdk` is simple: `from bunny_cdn_sdk import CoreClient, StorageClient, BunnyAPIError`. All public imports go through `__init__.py`. This plan sets up those re-exports so Phase 3 can implement the service clients without worrying about `__init__.py`.

**Output**

- `src/bunny_cdn_sdk/__init__.py` with all public re-exports
- INFRA-10 requirement implemented

**Full example structure**:
```python
"""Bunny CDN Python SDK.

A thin, typed Python SDK wrapping Bunny CDN REST APIs.
"""

from typing import TYPE_CHECKING

from bunny_cdn_sdk._exceptions import (
    BunnyAPIError,
    BunnyAuthenticationError,
    BunnyConnectionError,
    BunnyNotFoundError,
    BunnyRateLimitError,
    BunnySDKError,
    BunnyServerError,
    BunnyTimeoutError,
)

if TYPE_CHECKING:
    from bunny_cdn_sdk.core import CoreClient
    from bunny_cdn_sdk.storage import StorageClient

__all__ = [
    "BunnyAPIError",
    "BunnyAuthenticationError",
    "BunnyConnectionError",
    "BunnyNotFoundError",
    "BunnyRateLimitError",
    "BunnySDKError",
    "BunnyServerError",
    "BunnyTimeoutError",
    "CoreClient",
    "StorageClient",
]
```

**Why TYPE_CHECKING?** 
- CoreClient and StorageClient don't exist yet (Phase 1 created only stubs)
- Without TYPE_CHECKING, the import would raise ImportError at runtime
- With TYPE_CHECKING, the names appear in __all__ and type checkers understand them, but Python skips the import at runtime
- When Phase 3 implements these classes, they will be assignable to the same name (the stub modules already exist)

**Notes**:
- Imports are sorted by module name (exceptions before core/storage)
- Within exceptions, names are alphabetically sorted
- `__all__` is strictly alphabetical (not grouped)
- This will work even though CoreClient and StorageClient don't yet exist in their module stubs
