### Pagination Pattern

The Bunny Core API returns paginated responses in this envelope:
```json
{
  "Items": [...],
  "CurrentPage": 1,
  "TotalItems": 100,
  "HasMoreItems": true
}
```

Service clients will provide a callback that fetches a single page:
```python
async def fetch_page(page: int) -> PaginatedResponse:
    return await client._request('GET', f'/api/endpoint?page={page}')
```

The `pagination_iterator` will handle:
- Calling the fetch callback for page 1
- Checking `HasMoreItems` to determine if more pages exist
- Automatically fetching subsequent pages
- Yielding individual items from the `Items` array

<task type="auto">
  <name>Task 2: Implement pagination_iterator callback in _pagination.py</name>
  <files>src/bunny_cdn_sdk/_pagination.py</files>
  <action>
Implement the pagination iterator that service clients will use for both `list_*()` and `iter_*()` operations.

1. **Define the callback protocol** (type hint for fetch_page parameter):
   ```python
   from collections.abc import Callable
   from typing import TypeVar, AsyncIterator
   
   T = TypeVar('T')  # Item type
   FetchPage = Callable[[int], AsyncIterator[dict]]  # fetch_page(page_number) -> awaitable returning response dict
   ```

2. **Implement async_pagination_iterator**:
   - Signature: `async def pagination_iterator(fetch_page: Callable[[int], Awaitable[PaginatedResponse]], start_page: int = 1) -> AsyncIterator[dict]`
   - Logic:
     - Call `response = await fetch_page(start_page)` to fetch page 1
     - Yield each item from `response['Items']`
     - While `response['HasMoreItems']` is True:
       - Call `response = await fetch_page(start_page + 1)` (increment page for next fetch)
       - Yield each item from the new response's `Items`
     - When `HasMoreItems` is False, stop

3. **Implement list_single_page helper** (for service clients to use):
   - Signature: `async def list_single_page(fetch_page: Callable[[int], Awaitable[PaginatedResponse]], page: int = 1) -> PaginatedResponse`
   - Simply call and return: `return await fetch_page(page)`
   - This is the "raw response" operation (no pagination magic)

4. **Module docstring**:
   - Keep existing: `"""Pagination helpers (populated in Phase 2)."""`

5. **Exports**:
   - Add `__all__ = ['pagination_iterator', 'list_single_page']`

**Implementation notes**:
- `pagination_iterator` is async (yields via async for)
- `list_single_page` is async (returns a coroutine)
- Service clients will wrap these in sync functions via `asyncio.run()` in Phase 3
- The callback `fetch_page` returns a PaginatedResponse dict, not a coroutine object; callers await it

**Edge cases**:
- If `fetch_page` returns an empty Items list, still yield nothing (the while loop will check HasMoreItems and stop)
- If start_page > 1, the iterator respects it (for pagination resumption, though Phase 3 won't use this)

**Type hints**:
- Use `from typing import Awaitable` for type of `fetch_page` return
- Use `AsyncIterator[dict]` for the async generator return type
