# Plan 02-01: Base Client Infrastructure

**Objective**

Implement the `_BaseClient` class in `src/bunny_cdn_sdk/_client.py` that wraps `httpx.AsyncClient` with:
- Async internals (httpx), sync public surface (asyncio.run)
- `_request()` chokepoint with auth and User-Agent injection, HTTP status → exception mapping
- `_gather()` for concurrent batch operations
- Context manager support (sync and async)
- Optional injectable httpx.AsyncClient for testing

**Purpose**

The base client is the foundation for all service clients (Core, Storage). Every outbound request must carry proper authentication and error handling. This plan delivers all machinery before service clients are written.

**Output**

- `src/bunny_cdn_sdk/_client.py` with `_BaseClient` class
- All INFRA-01 through INFRA-05 requirements implemented

Create the `_BaseClient` class with the following skeleton and interfaces:

1. **Constructor**:
   - `__init__(self, api_key: str, client: httpx.AsyncClient | None = None)`
   - Store `api_key` as instance attribute
   - If `client` is provided, use it; otherwise create internal `self._client = httpx.AsyncClient()` and set `self._client_owner = True` (to track ownership for cleanup)
   - If `client` is provided, set `self._client_owner = False`

2. **Async context manager**:
   - `async def __aenter__(self) -> Self` — return self after ensuring client is open (no-op if using injected client)
   - `async def __aexit__(self, exc_type, exc_val, exc_tb) -> None` — close the client if we own it (self._client_owner is True)

3. **Sync context manager**:
   - `def __enter__(self) -> Self` — call `asyncio.run(self.__aenter__)` and return self
   - `def __exit__(self, exc_type, exc_val, exc_tb) -> None` — call `asyncio.run(self.__aexit__(exc_type, exc_val, exc_tb))`

4. **Async request method**:
   - `async def _request(self, method: str, url: str, **kwargs) -> httpx.Response`
   - Inject headers into kwargs: `headers = {**kwargs.get('headers', {}), 'AccessKey': self.api_key, 'User-Agent': 'bunny-cdn-sdk/0.1.0'}`
   - Call `self._client.request(method, url, headers=headers, **kwargs)` and return the response

5. **Async gather method**:
   - `async def _gather(self, *coroutines) -> list` — use `asyncio.gather(*coroutines)` and return results as list

6. **Sync wrapper for requests** (will be used by service clients):
   - `def _sync_request(self, method: str, url: str, **kwargs) -> httpx.Response` — call `asyncio.run(self._request(method, url, **kwargs))`

Do NOT implement error mapping yet (Task 2). Do NOT handle exceptions in this task.

<task type="auto">
  <name>Task 2: Implement HTTP status → exception mapping in _request() chokepoint</name>
  <files>src/bunny_cdn_sdk/_client.py</files>
  <action>
Modify the `_request()` method to catch HTTP errors and map them to the exception hierarchy.

1. **Wrap the actual request call**:
   - Call `self._client.request(method, url, headers=headers, **kwargs)` to get `response`
   - Use `response.raise_for_status()` to trigger on 4xx/5xx — this will raise `httpx.HTTPStatusError`
   - Return the `response` object if successful (2xx/3xx)

2. **Catch httpx.HTTPStatusError and map to BunnyAPIError subclasses**:
   - Extract `status_code = exc.response.status_code`
   - Extract `response = exc.response`
   - Attempt to parse error message from response body: `message = response.json().get('Message') or response.text or f'HTTP {status_code}'`
   - Map status codes:
     - 401 → raise `BunnyAuthenticationError(status_code, message, response)`
     - 404 → raise `BunnyNotFoundError(status_code, message, response)`
     - 429 → raise `BunnyRateLimitError(status_code, message, response)`
     - 5xx → raise `BunnyServerError(status_code, message, response)`
     - All others → raise generic `BunnyAPIError(status_code, message, response)`

3. **Catch httpx.ConnectError and httpx.ConnectTimeout**:
   - `httpx.ConnectTimeout` → raise `BunnyTimeoutError(f'Connection timeout: {exc}')`
   - `httpx.ConnectError` (any other connection error) → raise `BunnyConnectionError(f'Connection error: {exc}')`

4. **Catch httpx.TimeoutException** (request/read timeout):
   - Raise `BunnyTimeoutError(f'Request timeout: {exc}')`

5. **Do NOT catch and re-raise other httpx exceptions** — let them propagate as-is.

**Example error message extraction**: If the response JSON has a key `Message` (Bunny's common pattern), use that; fall back to response text, then fallback to generic HTTP status message.

## Threat Model

| Boundary | Description |
|----------|-------------|
| API Key Injection | AccessKey header crosses client→API boundary (untrusted network) |
| Error Response Parsing | Untrusted error messages from API responses enter exception objects |
| Concurrent Requests | Multiple coroutines share httpx.AsyncClient state |

## STRIDE Register

| Threat ID | Category | Component | Disposition | Mitigation |
|-----------|----------|-----------|-------------|-----------|
| T-02-01 | Information Disclosure | API Key in User-Agent | Accept | User-Agent does not contain key; only AccessKey header does (encrypted by HTTPS) |
| T-02-02 | Tampering | Error message injection in exception | Mitigate | Error messages from response body are parsed safely; no code execution via exception attributes |
| T-02-03 | Denial of Service | Concurrent gather with no limits | Mitigate | _gather() passes coroutines directly to asyncio.gather; caller controls concurrency |
